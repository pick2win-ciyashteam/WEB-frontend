📑 4. Lineouts & UCT Access Flow

Users can access the 📋 Lineouts Tab, which displays supported football matches, lineup release status, and UCT workflow availability.

⏳ Waiting for lineup. Before official football lineups are released:

  • users can view upcoming matches
  • UCT workflow remains unavailable
  • Status shown: ⏳ WAITING FOR LINEUP · 🔒 UCT Locked

▶️ Run UCT availability. Once official football lineups are released, Playing XI becomes visible, substitute players become visible, and the ▶️ Run UCT button becomes active. When users click ▶️ Run UCT, the structured UCT workflow sequence begins.

⚙️ 5. UCT Workflow Screens

🧩 UI Screen 1 — Substitute Player Configuration. Users can view 🟢 Playing XI players and 🔄 substitute players. Users may optionally select 0–3 substitute players. Substitute player selection is optional and completely user-controlled.

Substitute rules:

  • ✅ Mandate YES (M-YES)
  • ⚪ NA

Substitute players cannot be ❌ Mandate NO (M-NO).

🎯 UI Screen 2 — Mandate Player Configuration. Mandate configuration is optional, user-controlled, and preference-based.

  • ✅ Mandate YES (M-YES): select 1–2 players — selected players appear in all generated teams
  • ❌ Mandate NO (M-NO): select 1–2 players — selected players appear in 0 generated teams · 🚫 M-NO players cannot appear in Captain or Vice-Captain roles
  • ⚪ NA (No Mandate): no forced inclusion, no forced exclusion · participation depends on generation logic, combinations, category balancing, and configuration structures

Valid configurations: ✔ M-YES only · ✔ M-NO only · ✔ M-YES + M-NO · ✔ NA only

Invalid configurations: ❌ M-YES + M-NO + NA together · ❌ NA combined with M-YES or M-NO · ❌ more than 2 players per mandate type

👑 UI Screen 3 — Captain & Vice-Captain Configuration. This screen is mandatory. Users must select captaincy mode and select eligible players. Without valid configuration, 🚫 workflow cannot proceed.

🔄 CVC Mode: select 2–4 players; selected players rotate as 👑 Captain and ⭐ Vice-Captain.
